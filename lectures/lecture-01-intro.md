## 1. Что такое машинное обучение

Если очень коротко, машинное обучение — это способ извлекать знания из данных.

У нас есть гипотеза: в данных, которые мы собрали, есть некоторая скрытая информация. Мы хотим эту информацию извлечь и использовать для предсказаний или принятия решений.

В классическом программировании мы обычно пишем правила сами:

```text
данные + правила → результат
```

В машинном обучении подход другой:

```text
данные + правильные ответы → модель
модель + новые данные → предсказание
```

То есть мы не задаём все правила вручную, а обучаем модель находить закономерности по примерам.

Например:

* по данным о студентах хотим предсказывать итоговую оценку;
* по данным о клиентах хотим предсказывать отток;
* по изображению хотим определить, что на нём находится;
* по тексту письма хотим определить, спам это или нет.

Главная идея: раньше люди сами пытались выводить закономерности из наблюдений, а теперь мы хотим, чтобы часть этой работы делала машина.

---

## 2. Датасет, наблюдения и признаки

В машинном обучении мы почти всегда работаем с датасетом.

**Датасет** — это набор данных, состоящий из наблюдений.

**Наблюдение**, или **объект**, — это один элемент выборки.

Например:

* один студент;
* один клиент;
* одна квартира;
* одно письмо;
* одно изображение;
* одна транзакция.

Каждый объект описывается признаками.

**Признак** — это любая информация об объекте, которую мы можем использовать.

Примеры признаков:

| Объект      | Признаки                                          |
| ----------- | ------------------------------------------------- |
| студент     | посещаемость, оценки за домашки, оценка за Python |
| квартира    | площадь, район, этаж, год постройки               |
| клиент      | количество покупок, средний чек, активность       |
| письмо      | текст, отправитель, тема                          |
| изображение | пиксели или признаки, извлечённые из изображения  |

В ML часто используют английское слово **feature**, поэтому признаки часто называют **фичами**.

---

## 3. IID-предположение

Очень часто в машинном обучении предполагается, что данные являются IID.

```text
IID = independent and identically distributed
```

То есть объекты в выборке считаются:

1. **независимыми**;
2. **одинаково распределёнными**.

### Независимость

Независимость означает, что один объект не должен зависеть от другого.

Например, если в датасете один и тот же студент случайно оказался дважды, это может нарушать независимость наблюдений.

### Одинаковое распределение

Одинаковое распределение означает, что все объекты должны быть получены из одного и того же процесса.

Например, если мы собрали датасет животных Южной Африки, а потом случайно добавили туда белого медведя, то белый медведь, скорее всего, не относится к той же генеральной совокупности, что и остальные объекты.

В реальных данных IID почти никогда не выполняется идеально. Но это важное базовое предположение, на котором строится много методов и теоретических результатов.

---

## 4. Числовые и категориальные признаки

Признаки бывают разных типов.

### Числовые признаки

Числовые признаки принимают числовые значения.

Примеры:

* возраст;
* зарплата;
* площадь квартиры;
* температура;
* количество часов подготовки;
* оценка за домашнее задание.

Числовой признак обычно можно сравнивать, складывать, усреднять.

Например:

```text
площадь 40 м² < площадь 60 м²
```

---

### Категориальные признаки

Категориальные признаки принимают значения из конечного набора категорий.

Примеры:

* цвет глаз;
* город;
* тип устройства;
* категория товара;
* профессия;
* язык пользователя.

Например, цвет глаз может быть:

```text
карие, зелёные, голубые
```

У категориальных признаков обычно нет естественного порядка.

Нельзя сказать, что:

```text
карие < зелёные < голубые
```

Если мы просто закодируем категории числами:

```text
карие = 1
зелёные = 2
голубые = 3
```

то модель может ошибочно решить, что между ними есть порядок и расстояние.

Поэтому часто используют one-hot encoding.

Пример:

| Цвет глаз | is_brown | is_green | is_blue |
| --------- | -------: | -------: | ------: |
| карие     |        1 |        0 |       0 |
| зелёные   |        0 |        1 |       0 |
| голубые   |        0 |        0 |       1 |

Так один категориальный признак превращается в несколько бинарных признаков.

---

## 5. Матрица признаков

Если собрать все объекты и все признаки в одну таблицу, получится **матрица признаков**.

Её часто обозначают как `X`.

```text
X — матрица признаков
```

В этой матрице:

```text
строки — объекты
столбцы — признаки
```

Пример:

| Студент | Python score | Statistics score | Attendance |
| ------- | -----------: | ---------------: | ---------: |
| A       |            8 |                7 |       0.90 |
| B       |            5 |                4 |       0.50 |
| C       |            9 |                8 |       0.80 |

Если убрать колонку с именем студента, получим матрицу признаков:

```text
X = [
  [8, 7, 0.90],
  [5, 4, 0.50],
  [9, 8, 0.80]
]
```

Каждый объект можно представить как вектор в пространстве признаков.

Например, первый студент:

```text
x = [8, 7, 0.90]
```

Если у объекта `p` признаков, то объект можно рассматривать как точку в `p`-мерном пространстве.

## 6. Таргет

В задачах обучения с учителем кроме признаков есть ещё целевая переменная.

Её называют:

```text
target
label
y
целевая переменная
правильный ответ
```

Пример:

| Python score | Statistics score | Attendance | Final grade |
| -----------: | ---------------: | ---------: | ----------: |
|            8 |                7 |       0.90 |           8 |
|            5 |                4 |       0.50 |           4 |
|            9 |                8 |       0.80 |           9 |

Здесь:

```text
X = признаки
y = итоговая оценка
```

Если мы хотим предсказывать итоговую оценку, то `Final grade` — это таргет.

## 7. Обучение с учителем и без учителя

В машинном обучении есть два больших режима:

```text
supervised learning
unsupervised learning
```

---

### Обучение с учителем

**Обучение с учителем** — это постановка, где у нас есть объекты и правильные ответы.

```text
X — признаки
y — правильные ответы
```

Примеры:

| Задача                    | Признаки                  | Таргет           |
| ------------------------- | ------------------------- | ---------------- |
| Предсказать цену квартиры | площадь, район, этаж      | цена             |
| Определить спам           | текст письма, отправитель | спам / не спам   |
| Предсказать отток клиента | активность, покупки       | уйдёт / не уйдёт |
| Предсказать оценку        | оценки за прошлые курсы   | итоговая оценка  |

В supervised learning модель учится на примерах, где правильный ответ уже известен.

---

### Обучение без учителя

**Обучение без учителя** — это постановка, где у нас есть только данные, но нет правильных ответов.

```text
X есть
y нет
```

Мы не знаем заранее, какой ответ должен быть у каждого объекта. Вместо этого мы хотим найти структуру в данных.

Примеры:

* разбить клиентов на группы;
* найти похожие товары;
* найти аномальные объекты;
* уменьшить размерность данных;
* визуализировать сложные данные.

Например, у нас есть данные о клиентах интернет-магазина, но нет заранее заданных классов. Мы можем попробовать выделить группы клиентов с похожим поведением.

---

## 8. Регрессия и классификация

В supervised learning есть две основные постановки:

```text
регрессия
классификация
```

---

### Регрессия

**Регрессия** — это задача, где таргет является числом.

Примеры:

* предсказать цену квартиры;
* предсказать температуру;
* предсказать количество заказов;
* предсказать время доставки;
* предсказать итоговую оценку студента.

Формально:

```text
y ∈ R
```

То есть `y` — некоторое число.

Иногда модель может предсказывать не одно число, а вектор чисел.

Например, если нужно предсказать цвет в формате RGB, нужно предсказать сразу три числа:

```text
R, G, B
```

---

### Классификация

**Классификация** — это задача, где таргет принимает одно из конечного числа значений.

Примеры:

* письмо: спам или не спам;
* клиент: уйдёт или не уйдёт;
* изображение: кошка, собака или машина;
* транзакция: мошенническая или обычная.

Формально:

```text
y ∈ {1, 2, ..., K}
```

где `K` — число классов.

Пример бинарной классификации:

```text
0 — не спам
1 — спам
```

Пример многоклассовой классификации:

```text
0 — кошка
1 — собака
2 — машина
3 — человек
```

Иногда регрессия и классификация могут быть близки. Например, оценку от 1 до 5 можно рассматривать и как число, и как категорию. Выбор зависит от постановки задачи.

---

## 9. Модель

Модель — это функция, которая отображает объект в предсказание.

```text
f(x) = y_pred
```

Например:

```text
признаки квартиры → предсказанная цена
признаки письма → вероятность спама
признаки студента → предсказанная оценка
```

Если у нас есть матрица признаков `X`, модель строит предсказания:

```text
f(X)
```

Цель обучения — выбрать такую модель, которая хорошо предсказывает ответы на новых объектах.

---

## 10. Функция потерь

Чтобы понять, насколько модель хорошая, нужна функция потерь.

**Функция потерь** показывает, насколько предсказание модели отличается от правильного ответа.

```text
L(y, f(x))
```

Где:

* `y` — правильный ответ;
* `f(x)` — предсказание модели;
* `L` — величина ошибки.

Для регрессии можно использовать квадратичную ошибку:

```text
L(y, f(x)) = (y - f(x))²
```

Для классификации можно использовать ошибку 0/1:

```text
L(y, f(x)) = 0, если класс угадан
L(y, f(x)) = 1, если класс не угадан
```

Функция потерь нужна, чтобы сравнивать модели между собой.

Например:

```text
модель A ошибается меньше, чем модель B
→ модель A лучше по выбранной функции потерь
```

Важно: пока мы не понимаем, как измерять качество, задача машинного обучения не поставлена полностью.

---

## 11. Формальная постановка ML-задачи

Чтобы поставить задачу машинного обучения, нужно определить три вещи:

1. данные;
2. модель;
3. функцию потерь.

Дано:

```text
X — матрица признаков
y — вектор правильных ответов
```

Нужно найти модель `f`, которая минимизирует ошибку:

```text
min L(y, f(X))
```

В реальных задачах сначала нужно понять, что именно мы оптимизируем.

Например, фраза:

```text
Хотим предсказывать отток клиентов
```

ещё не является полной постановкой.

Нужно уточнить:

* что такое отток;
* за какой период мы его предсказываем;
* какие данные доступны в момент предсказания;
* какую метрику качества используем;
* какие ошибки дороже: false positive или false negative.

Без функции потерь или метрики мы не можем строго сказать, какая модель лучше.

---

## 12. Параметры модели

У модели могут быть параметры.

Например, модель может предсказывать оценку студента как взвешенную сумму признаков:

```text
predicted_grade = w1 * python_score + w2 * statistics_score
```

Здесь `w1` и `w2` — параметры модели.

Параметры подбираются в процессе обучения.

То есть модель учится находить такие значения параметров, при которых ошибка становится меньше.

---

## 13. Гиперпараметры

Гиперпараметры — это настройки, которые не подбираются самой моделью во время обучения, а задаются заранее.

Примеры:

* количество соседей `k` в kNN;
* глубина дерева решений;
* коэффициент регуляризации;
* learning rate;
* количество деревьев в ансамбле;
* выбор метрики расстояния.

Главное отличие:

```text
параметры обучаются моделью
гиперпараметры выбираются до обучения или подбираются отдельной процедурой
```

Например, в kNN число соседей `k` — это гиперпараметр. Модель не узнаёт его сама из данных, мы должны выбрать его заранее или подобрать на validation set.

---

## 14. Метод максимального правдоподобия

Метод максимального правдоподобия — один из базовых способов оценивать параметры модели.

Идея: мы хотим выбрать такие параметры, при которых наблюдаемые данные наиболее правдоподобны.

Пусть у нас есть выборка:

```text
x1, x2, ..., xn
```

И мы считаем, что данные были порождены некоторым распределением с параметром `θ`.

Правдоподобие:

```text
L(θ) = P(data | θ)
```

Это вероятность получить наблюдаемые данные при условии, что параметр равен `θ`.

Важно не путать:

```text
P(data | θ) — правдоподобие
P(θ | data) — вероятность параметра при условии данных
```

В методе максимального правдоподобия мы ищем:

```text
θ* = argmax P(data | θ)
```

То есть выбираем такой параметр, при котором наша выборка выглядит наиболее вероятной.

---

## 15. Почему используют логарифм правдоподобия

Если объекты независимы, вероятность всей выборки можно разложить в произведение:

```text
P(data | θ) = P(x1 | θ) * P(x2 | θ) * ... * P(xn | θ)
```

Но произведение большого числа вероятностей неудобно:

* оно может быть очень маленьким;
* с ним сложнее работать аналитически;
* производные от произведений считать неудобно.

Поэтому часто берут логарифм:

```text
log L(θ) = log P(x1 | θ) + log P(x2 | θ) + ... + log P(xn | θ)
```

Логарифм превращает произведение в сумму.

Максимум функции и максимум её логарифма достигаются в одной и той же точке, потому что логарифм — монотонное преобразование.

---

## 16. Теорема Байеса

Теорема Байеса связывает две условные вероятности:

```text
P(A | B) = P(B | A) * P(A) / P(B)
```

В задачах классификации нас интересует вероятность класса при условии объекта:

```text
P(y = c | x)
```

То есть:

```text
какова вероятность, что объект x принадлежит классу c?
```

По теореме Байеса:

```text
P(y = c | x) = P(x | y = c) * P(y = c) / P(x)
```

Где:

* `P(y = c | x)` — вероятность класса при условии объекта;
* `P(x | y = c)` — вероятность увидеть такой объект внутри класса `c`;
* `P(y = c)` — априорная вероятность класса;
* `P(x)` — вероятность объекта.

Для классификации нужно выбрать самый вероятный класс:

```text
c* = argmax P(y = c | x)
```

Подставим Байеса:

```text
c* = argmax P(x | y = c) * P(y = c) / P(x)
```

Так как `P(x)` одинаково для всех классов, при выборе максимума его можно не учитывать:

```text
c* = argmax P(x | y = c) * P(y = c)
```

## 17. Naive Bayes

Naive Bayes — это байесовский классификатор с наивным предположением.

Пусть объект описывается признаками:

```text
x = (x1, x2, ..., xp)
```

В общем случае вероятность `P(x | y = c)` сложно оценить, потому что `x` — это целый вектор признаков.

Naive Bayes делает предположение:

```text
признаки независимы друг от друга при условии класса
```

Тогда вероятность вектора признаков можно разложить:

```text
P(x | y = c) = P(x1 | y = c) * P(x2 | y = c) * ... * P(xp | y = c)
```

И классификатор выбирает класс:

```text
c* = argmax P(y = c) * P(x1 | y = c) * ... * P(xp | y = c)
```

Это предположение называется наивным, потому что в реальных данных признаки почти никогда не являются полностью независимыми.

Например, в данных о человеке рост и вес могут быть зависимы. В текстах слова тоже часто зависят друг от друга.

Но несмотря на это, Naive Bayes часто работает неплохо как простая базовая модель.

---

## 18. Как оценивать вероятности в Naive Bayes

### Априорная вероятность класса

`P(y = c)` можно оценить по частоте класса в обучающей выборке.

Например, если в датасете 1000 писем, из них 300 — спам, то:

```text
P(y = spam) = 300 / 1000 = 0.3
```

---

### Категориальные признаки

Если признаки категориальные, можно считать частоты значений признаков внутри каждого класса.

Например:

```text
P(word = "sale" | y = spam)
P(word = "sale" | y = not_spam)
```

Если слово часто встречается в спаме и редко встречается в обычных письмах, оно будет увеличивать вероятность класса `spam`.

---

### Числовые признаки

Если признаки числовые, можно предположить, что внутри каждого класса они имеют некоторое распределение.

Например, можно считать, что признак распределён нормально.

Тогда для каждого класса и каждого признака оцениваются:

```text
среднее
дисперсия
```

Такой вариант называется Gaussian Naive Bayes.

---

## 19. Зачем нужен Naive Bayes

Naive Bayes полезен как простая первая модель.

Плюсы:

* очень простой;
* быстрый;
* легко реализуется;
* хорошо работает как baseline;
* часто хорошо работает на текстовых данных;
* не требует сложной оптимизации.

Минусы:

* делает сильное предположение о независимости признаков;
* может плохо работать при сильной зависимости признаков;
* вероятности могут быть плохо откалиброваны;
* обычно уступает более сложным моделям на сложных задачах.

---

## 20. kNN: метод k ближайших соседей

kNN — ещё один простой базовый алгоритм машинного обучения.

Идея:

```text
похожие объекты обычно имеют похожие ответы
```

Если новый объект похож на объекты некоторого класса, можно отнести его к этому классу.

Метод k ближайших соседей не строит сложную параметрическую модель. Он хранит обучающую выборку и использует её на этапе предсказания.

---

## 21. kNN для классификации

Пусть есть обучающая выборка:

```text
X_train, y_train
```

И новый объект:

```text
x_new
```

Алгоритм:

1. Посчитать расстояние от `x_new` до всех объектов обучающей выборки.
2. Выбрать `k` ближайших объектов.
3. Посмотреть их классы.
4. Выбрать самый частый класс среди соседей.
5. Вернуть этот класс как предсказание.

Пример:

```text
k = 5

среди 5 ближайших соседей:
3 объекта класса A
2 объекта класса B

предсказание: класс A
```

---

## 22. kNN для регрессии

kNN можно использовать не только для классификации, но и для регрессии.

Алгоритм почти такой же:

1. Посчитать расстояние до объектов обучающей выборки.
2. Выбрать `k` ближайших соседей.
3. Усреднить их целевые значения.

Пример:

```text
цены ближайших квартир:
10 млн, 11 млн, 12 млн

предсказание:
11 млн
```

Можно использовать обычное среднее или взвешенное среднее.

Во взвешенном варианте ближайшие соседи имеют больший вес.

---

## 23. Расстояния в kNN

kNN требует определить, что значит “близко”.

Для этого нужна метрика расстояния.

Самый популярный вариант — евклидово расстояние.

Для двух объектов:

```text
x = (x1, x2)
z = (z1, z2)
```

евклидово расстояние:

```text
distance(x, z) = sqrt((x1 - z1)^2 + (x2 - z2)^2)
```

Другие варианты:

* Manhattan distance;
* Minkowski distance;
* cosine distance;
* Hamming distance.

Выбор расстояния — это гиперпараметр.

---

## 24. Гиперпараметр k

Количество соседей `k` — главный гиперпараметр kNN.

Если `k = 1`, модель смотрит только на одного ближайшего соседа.

Плюсы маленького `k`:

* модель гибкая;
* хорошо подстраивается под train.

Минусы маленького `k`:

* чувствительность к шуму;
* чувствительность к выбросам;
* риск переобучения.

Если `k` большое, модель смотрит на много соседей.

Плюсы большого `k`:

* предсказания устойчивее;
* меньше влияние отдельных шумных объектов.

Минусы большого `k`:

* модель становится слишком грубой;
* можно потерять локальную структуру данных;
* риск недообучения.

Обычно `k` подбирают на validation set или с помощью кросс-валидации.

---

## 25. Масштабирование признаков для kNN

kNN основан на расстояниях, поэтому масштаб признаков очень важен.

Пример:

```text
признак 1: рост в метрах, значения от 1.5 до 2.0
признак 2: зарплата в рублях, значения от 50 000 до 300 000
```

Если считать евклидово расстояние без масштабирования, зарплата будет почти полностью определять расстояние между объектами, а рост почти не будет влиять.

Поэтому перед kNN признаки обычно масштабируют.

### Standard scaling

```text
x_scaled = (x - mean) / std
```

После такого преобразования среднее становится равным 0, а стандартное отклонение — 1.

### MinMax scaling

```text
x_scaled = (x - min) / (max - min)
```

После такого преобразования значения попадают в диапазон от 0 до 1.

Важно: параметры масштабирования считаются только на train, а потом применяются к validation и test.

---

## 26. Train, validation и test

Чтобы честно оценить качество модели, нельзя проверять её только на тех данных, на которых она обучалась.

Модель могла просто запомнить ответы.

Поэтому данные делят на части:

```text
train — данные для обучения
validation — данные для подбора гиперпараметров
test — данные для финальной оценки
```

Например:

```text
70% — train
15% — validation
15% — test
```

Для kNN на validation можно подбирать:

* `k`;
* метрику расстояния;
* использовать веса или нет;
* способ масштабирования.

Test лучше использовать только один раз — для финальной оценки выбранной модели.

---

## 27. Что важно запомнить

Машинное обучение — это способ извлекать закономерности из данных и использовать их для предсказаний.

Базовые сущности:

```text
dataset — набор данных
object / observation — объект
feature — признак
target / label — целевая переменная
design matrix — матрица признаков
model — модель
loss function — функция потерь
```

Supervised learning — это обучение с правильными ответами.

Unsupervised learning — это поиск структуры в данных без правильных ответов.

Регрессия предсказывает число.

Классификация предсказывает класс.

Naive Bayes использует теорему Байеса и предположение о независимости признаков при условии класса.

kNN предсказывает ответ по ближайшим объектам из обучающей выборки.

Для kNN критически важно масштабировать признаки, потому что алгоритм основан на расстояниях.
